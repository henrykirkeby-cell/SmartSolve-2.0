  STEP 1: SET UP STRIPE (your payment system)
━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━

1. Go to https://dashboard.stripe.com
2. Sign in with your existing Stripe account

3. CREATE A PRODUCT & PRICE:
   - Click "Products" in the left sidebar
   - Click "+ Add product"
   - Name: "McGraw Hill Auto Answer"
   - Add a price: click "Add a price"
     - Pricing model: Recurring
     - Price: e.g. $9.99
     - Billing period: Monthly
   - Click "Save product"
   - Copy the "Price ID" (looks like price_1AbcXXXXXXXX)
   - Paste it into your .env file as STRIPE_PRICE_ID

4. CREATE A PAYMENT LINK (for customers to buy):
   - Click "Payment Links" in sidebar
   - Click "+ New"
   - Select your product/price
   - Click "Create link"
   - Copy the URL (looks like https://buy.stripe.com/XXXXX)
   - Paste it into popup.js as STRIPE_PAYMENT_LINK

5. GET YOUR API KEYS:
   - Click "Developers" → "API keys"
   - Copy "Secret key" (starts with sk_live_...)
   - Paste it into .env as STRIPE_SECRET_KEY


━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━
  STEP 2: DEPLOY YOUR BACKEND SERVER (Railway)
━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━

Railway is free to start and super easy. It hosts your server 24/7.

1. Go to https://railway.app and sign up (free)

2. PUSH YOUR CODE TO GITHUB:
   - Go to https://github.com and create a new repository
   - Name it "mcgrawhill-auto-answer-backend"
   - Make it Private
   - Upload all files from the "mcgrawhill-backend" folder to it
   - (Do NOT upload your .env file — that stays local and secret)

3. DEPLOY ON RAILWAY:
   - On Railway, click "New Project"
   - Click "Deploy from GitHub repo"
   - Select your "mcgrawhill-auto-answer-backend" repo
   - Railway will detect it's a Node.js app and deploy automatically!

4. ADD YOUR ENVIRONMENT VARIABLES:
   - In Railway, click your project → "Variables" tab
   - Add each variable from your .env file:
     ANTHROPIC_API_KEY = sk-ant-...
     STRIPE_SECRET_KEY = sk_live_...
     STRIPE_PRICE_ID   = price_...
     EMAIL_USER        = yourname@gmail.com
     EMAIL_PASS        = your_gmail_app_password
     (Don't set PORT — Railway handles that automatically)

5. GET YOUR SERVER URL:
   - Click "Settings" tab in Railway
   - Under "Networking", click "Generate Domain"
   - You'll get a URL like: https://your-app.railway.app
   - Copy this URL!

6. UPDATE YOUR EXTENSION:
   - Open mcgrawhill-extension/background.js
   - Replace 'https://your-app-name.railway.app' with your real URL
   - Open manifest.json and do the same in host_permissions

7. SET UP STRIPE WEBHOOK (so Stripe notifies your server of payments):
   - In Stripe: Developers → Webhooks → "Add endpoint"
   - Endpoint URL: https://your-app.railway.app/webhook
   - Events to listen to:
       checkout.session.completed
       customer.subscription.deleted
       invoice.payment_failed
   - Click "Add endpoint"
   - Click "Reveal" under "Signing secret"
   - Copy it → add to Railway as STRIPE_WEBHOOK_SECRET

  STEP 3: SET UP GMAIL FOR SENDING LICENSE KEYS
━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━

Your server emails license keys to customers automatically.

1. Go to https://myaccount.google.com
2. Click "Security"
3. Under "How you sign in to Google", click "2-Step Verification"
   and enable it (required for App Passwords)
4. Go back to Security → search "App passwords"
5. Create a new App Password:
   - App name: "McGraw Hill Server"
   - Copy the 16-character password
6. In Railway Variables:
   EMAIL_USER = your gmail address
   EMAIL_PASS = the 16-character app password

  STEP 4: PUBLISH TO CHROME WEB STORE
━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━

One-time $5 developer fee, then free to publish.

BEFORE YOU SUBMIT, YOU NEED:
  □ Extension icons (PNG files, 3 sizes):
      icons/icon16.png   — 16x16 pixels
      icons/icon48.png   — 48x48 pixels
      icons/icon128.png  — 128x128 pixels
    → Make these at https://www.canva.com (free)
    → Use a robot/brain emoji on a green background

  □ At least 1 screenshot (1280x800 or 640x400 pixels)
    → Take a screenshot of the floating green tab on McGraw Hill

  □ A short description (132 characters max)
    Example: "Automatically answers McGraw Hill Connect questions using AI. Save hours — subscribe and let it do the work."

  □ A Privacy Policy URL (required since extension uses storage)
    → Easiest option: create a free page at https://www.privacypolicygenerator.info
      and host it on a free site like https://sites.google.com

  □ Your backend URL updated in background.js and manifest.json


PUBLISHING STEPS:

1. Go to https://chrome.google.com/webstore/devconsole
2. Pay the one-time $5 developer registration fee
3. Click "New Item"
4. ZIP your entire "mcgrawhill-extension" folder:
   - Select all files inside the folder
   - Right-click → "Compress" / "Send to ZIP"
   - Name it "mcgrawhill-extension.zip"
5. Upload the ZIP file
6. Fill in the store listing:
   - Name: McGraw Hill Auto Answer
   - Short description: (your 132 char description)
   - Category: Education
   - Upload your screenshots
   - Upload your icons
   - Paste your Privacy Policy URL
7. Under "Permissions", explain why you need each permission:
   - activeTab: "To read the current McGraw Hill question"
   - storage: "To save the user's license key locally"
8. Click "Submit for Review"

⏱ REVIEW TIME: Usually 1-3 business days for new extensions.
   You'll get an email when it's approved.

  HOW MONEY FLOWS (summary)
━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━

  Customer clicks "Get Subscription" in extension popup
       ↓
  Pays you $9.99/month on your Stripe payment link
       ↓
  Stripe sends a webhook to your Railway server
       ↓
  Server generates a license key + emails it to the customer
       ↓
  Customer enters the key in the extension
       ↓
  Extension works — questions get answered automatically
       ↓
  Stripe auto-charges them each month
  If they cancel → server deactivates their key automatically
